# 产品需求文档 (PRD) - 校园二手书交易平台「书窝」

## 1. 产品概述

「书窝」是一个面向高校学生的校园二手书交易平台，解决毕业季卖书广告刷屏微信群、信息难找、交易流程混乱的痛点。卖家通过扫码填写书名、作者、成色、定价即可快速上架；买家按专业课程标签筛选书籍，看中直接下单，订单全程跟踪物流，交易完成后双方互评。平台还内置消息模块支持讨价还价私聊，个人中心集中管理在售书籍与成交记录。
- 目标用户：在校大学生（兼顾买卖双方身份）
- 核心价值：让校园闲置书籍流转有序、可追溯、有温度

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 学生用户 | 学号/手机号注册 | 浏览、上架、下单、私聊、评价、管理个人中心 |

### 2.2 功能模块

1. **首页**：顶部导航 + 书签式分类标签 + 书籍瀑布流卡片 + 搜索栏
2. **上架页**：扫码录入 + 书籍信息表单（书名/作者/成色/定价/专业课程标签）
3. **书籍详情页**：书封大图 + 书籍信息 + 卖家信息 + 立即下单 + 私聊卖家
4. **订单页**：订单状态时间轴 + 物流跟踪 + 确认收货
5. **消息页**：会话列表 + 聊天窗口（讨价还价）
6. **个人中心**：在售书籍 + 成交记录 + 我的评价 + 账户设置

## 3. 核心流程

### 3.1 上架流程
卖家进入上架页 → 扫码录入 ISBN → 系统自动回填书名作者 → 补充成色/定价/专业课程标签/书封 → 提交上架 → 书籍出现在首页瀑布流。

### 3.2 购买流程
买家在首页按书签标签筛选 → 浏览瀑布流卡片 → 点击进入详情页 → 立即下单/先私聊讨价 → 生成订单 → 卖家发货填物流 → 买家跟踪物流 → 确认收货 → 双方互评。

### 3.3 评价流程
订单确认收货后 → 触发评价入口 → 买家对卖家评分留言 / 卖家对买家评分留言 → 评价计入信用分。

## 4. 用户界面设计

### 4.1 设计风格

学院清新风 —— 像一本翻开的旧书，温润、克制、带点书卷气。

- **主色**：墨绿 `#2E7D32`（书脊的稳重），辅以深墨绿 `#1B5E20`、嫩芽绿 `#66BB6A` 点缀
- **底色**：米白 `#FAF7F0`（旧书纸页的暖白），卡片用纯白 `#FFFFFF`
- **文字**：墨黑 `#212121`、灰墨 `#6B6B6B`
- **强调色**：暖橘 `#E65100`（用于价格、未读提示，少量点缀）
- **按钮风格**：圆角矩形（8px），主按钮墨绿实底带柔和投影，次按钮线框描边
- **字体**：标题用衬线字体（思源宋体 / Noto Serif SC）体现书卷气，正文用黑体（思源黑体 / Noto Sans SC）保证可读性
- **布局风格**：顶部固定导航 + 左侧无 / 主体瀑布流卡片网格 + 底部留白
- **图标/Emoji**：使用线性书卷主题图标（书、书签、钢笔、信封）
- **分类标签**：做成书签样子，顶部圆角、底部带 V 型缺口或丝带垂坠感，选中态墨绿填充
- **卡片交互**：书封卡片 hover 时上浮 8px、阴影加深、书封微倾斜
- **纹理细节**：米白底叠加极淡的纸张噪点纹理，营造旧书氛围

### 4.3 响应式

- 桌面优先（1280px+），书封瀑布流双列展示
- 平板（768px~1280px）瀑布流保持双列，导航简化
- 移动端（<768px）单列瀑布流，导航折叠为汉堡菜单，消息页单栏切换

### 4.4 动效

- 页面加载：书封卡片交错淡入上浮（staggered reveal）
- 卡片 hover：transform translateY(-8px) + 阴影加深 + 书脊高光
- 分类切换：瀑布流重排过渡
- 时间轴：状态推进时节点脉冲动画
- 消息气泡：从底部滑入淡出
